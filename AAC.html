ARD SAFE project

Project made using Tinkercad

Project Overview

The ARD-SAFE project is an intelligent safe system that integrates two-factor authentication through a distance sensor and a PIN code. Designed to ensure a high level of security and flexibility, this solution effectively protects personal and professional assets by utilizing microcontrolled platforms like Arduino to implement a functional and efficient system.

The system combines multiple sensors and input devices in a cohesive manner to ensure that only authorized users have access to the safe. This report details the journey from theoretical foundation to practical implementation, including component selection, electrical interconnections, software logic, and testing performed.

Key Features

  • Two-Factor Authentication: Combines PIN code verification with proximity detection
  • Persistent Memory: Safe configuration and PIN data remain stored even after power failures
  • Multiple Distance Intervals: Can be configured with various valid distance ranges
  • User Feedback: Provides clear instructions and status updates via LCD display
  • System Reset: Special commands allow for system reset and reconfiguration
  • Modular Architecture: Separate microcontrollers handle different aspects of the system

Implementation

The ARD SAFE project architecture consists of three main components:

1. System Architecture: The first Arduino (Master) manages the keypad and LCD, receiving user inputs and displaying feedback messages. The second Arduino (Slave) validates user proximity through the distance sensor and controls the relay module. The two boards communicate via UART protocol using RX/TX pins.

2. Hardware Assembly: Matrix keypad connected to digital pins 2-9 on Master Arduino. 16x2 LCD configured in 4-bit mode using pins 10-13 on Master Arduino. HC-SR04 ultrasonic sensor connected to Slave Arduino on pins 7 (Trigger) and 8 (Echo). Relay module connected to digital pin 9 on Slave Arduino.

3. Software Development:
The software consists of two main components:
1. Master Arduino Code: Handles keypad input, LCD communication, and sending commands to Slave Arduino

2. Slave Arduino Code: Processes received commands, validates distance sensor data, and controls the relay module

Key functions include:

  • setup(): Initializes communications and sets initial states
  • loop(): Continuously reads keypad input and processes user interactions
  • processInput(): Manages user input after collection
  • definePIN(): Configures and stores new PIN in EEPROM
  • verifyPIN(): Checks input PIN against stored value
  • getDistance(): Requests distance data from Slave Arduino
  • checkDistanceInAnyInterval(): Verifies if measured distance falls within defined ranges
  • configureInterval(): Allows users to define distance ranges for authorized access
  • resetSystem(): Clears stored PIN and all defined intervals
  • unlockSafe(): Activates relay to unlock the safe when authentication succeeds

    Memory Management

    Challenge:

    Limited SRAM and need for persistent data storage

    Solution:

    Utilized EEPROM for PIN storage and optimized code to minimize SRAM usage

    Reliable Distance Sensing

    Challenge:

    Achieving consistent distance measurements

    Solution:

    Implemented validation and error checking in the distance measurement function

    Secure Authentication

    Challenge:

    Creating a robust two-factor authentication system

    Solution:

    Combined PIN verification with configurable distance intervals for enhanced security

    Inter-Board Communication

    Challenge:

    Establishing reliable data exchange between Arduino boards

    Solution:

    Implemented UART protocol with proper error handling and timeouts

    Results & Impact

    The ARD-SAFE system demonstrated:

    - Efficient ultrasonic sensor response with an average validation time of 200ms

    - Stable relay module operation, enabling secure locking control

    - Effective UART communication between microcontrollers

    - Successful implementation of two-factor authentication

    - Potential for real-world applications in both domestic and commercial environments

    While the system achieved its initial objectives, further enhancements could include adding an extra authentication layer like RFID, or replacing UART with I2C for faster communication. Overall, the project resulted in a functional, secure, and customizable system with significant practical applications.
